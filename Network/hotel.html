Hotel Chain Network Design — Perth & Sydney

Scenario-based project covering VLANs, MST, OSPF multi-area, ACLs, NAT, PPP, and HSRP

Scenario & Requirements

A hotel chain has two major sites: Perth and Sydney. Each site contains 56 guests across two floors plus staff. The corporate network uses the private addressing space 10.0.0.0/8. The design must be scalable and secure. Key requirements are listed below.

Requirements

  • VLAN segmentation: VLAN 1000 = Staff, VLAN 20 = Guest Level 20, VLAN 3 = Guest Level 3 (student-based)
  • Trunking: 802.1Q on distribution→access links
  • MST spanning-tree: name HCG revision 1; VLAN→MST instance mapping described in configs
  • Inter-VLAN routing on distribution (L3) switches + OSPF multi-area:
    • Perth uses Area 1 (10.1.0.0/16)
    • Sydney uses Area 2 (10.2.0.0/16)
    • Perth↔Sydney backbone Area 0 (10.0.255.0/30)
  • Access Control:
    • Guests cannot access internal networks (10.0.0.0/8)
    • Block phishing IP range: 1.2.3.4–1.2.3.10
    • Guests allowed only HTTP, HTTPS, and ICMP
  • ISP connectivity: PPP serial links with CHAP, NAT pools, default route advertisement into OSPF
  • HSRP for Staff VLAN (1000) with virtual gateway 10.1.128.1 and preempt priorities

Key learning points

  • MST reduces STP zones while maintaining VLAN separation — root design balances load between PERDL1 & SYDDL2 as primary roots for different instances.
  • OSPF summarization reduces route table size at area boundaries (area-range commands provided).
  • ACLs are applied on VLAN SVI interfaces and on serial interfaces (outbound) to block phishing IPs and restrict guest protocols.
  • NAT overload allows many internal hosts to share a small public address pool; PPP with CHAP secures ISP authentication.
  • HSRP provides virtual gateway redundancy for Staff VLANs — set priorities and preempt so preferred device assumes active role.
